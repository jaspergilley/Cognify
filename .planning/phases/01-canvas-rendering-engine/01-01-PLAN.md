  <action>
    Build five standalone engine modules. Each module is a plain JS file (no React, no JSX) that exports factory functions. All modules must work independently and be testable without a DOM (except where browser APIs are needed).

    **src/engine/frameLoop.js** -- `createFrameLoop(onFrame, expectedFrameMs)`
    - Uses requestAnimationFrame for the tick loop
    - `onFrame` callback receives `{ frameCount, delta, timestamp, isDropped }`
    - `frameCount` increments by 1 each tick (never skips, never doubles)
    - `delta` calculated from rAF timestamp parameter (not performance.now() -- rAF provides its own high-res timestamp)
    - `isDropped` is true when `delta > expectedFrameMs * 1.5` (per research discretion recommendation)
    - Tracks dropped frame count and log internally via a frame monitor
    - Exports: `{ start(), stop(), isRunning(), getFrameCount(), getDroppedFrames(), getDroppedLog(), resetTiming(), reset() }`
    - `resetTiming()` sets lastTime to 0 so next frame computes a normal delta (used after visibility resume)
    - `reset()` resets frameCount, droppedCount, and log
    - `stop()` calls cancelAnimationFrame
    - CANV-02: Frame loop is rAF-only, no setTimeout/setInterval
    - CANV-09: Delta timing via timestamp comparison each frame, dropped frame detection at 1.5x threshold

    **src/engine/canvasScaler.js** -- `setupCanvasDPI(canvas, ctx)` and `calculateAspectRatio(containerWidth, containerHeight, targetRatio, minWidth, minHeight)`
    - `setupCanvasDPI`: reads `window.devicePixelRatio || 1`, gets `canvas.getBoundingClientRect()`, sets `canvas.width = Math.round(rect.width * dpr)`, `canvas.height = Math.round(rect.height * dpr)`, applies `ctx.scale(dpr, dpr)`. Returns `{ width: rect.width, height: rect.height, dpr }`.
    - `calculateAspectRatio`: given container dimensions, computes the largest 4:3 rectangle that fits. Enforces minimum dimensions (600x450 default). Returns `{ width, height, isBelowMinimum }`. `isBelowMinimum` is true when container cannot fit the minimum size.
    - CANV-01: DPI scaling via devicePixelRatio
    - CANV-08: All dimensions Math.round() to integers

    **src/engine/refreshRateDetector.js** -- `detectRefreshRate(sampleCount = 120, warmupFrames = 20)`
    - Returns a Promise that resolves with `{ hz, frameDurationMs }`
    - Collects `sampleCount` rAF timestamp intervals
    - Discards first `warmupFrames` intervals (page load jitter)
    - Sorts remaining intervals, takes median
    - Derives `hz = Math.round(1000 / median)`
    - If coefficient of variation > 0.2, fall back to `{ hz: 60, frameDurationMs: 16.667 }` and set `fallback: true` in the result
    - CANV-06: Refresh rate detection at startup

    **src/engine/visibilityManager.js** -- `createVisibilityManager(frameLoop)`
    - Listens for `document.visibilitychange`
    - On hidden: records whether frameLoop was running, calls `frameLoop.stop()`
    - On visible: if was running, calls `frameLoop.resetTiming()` then `frameLoop.start()`
    - Returns a cleanup function that removes the event listener
    - CANV-07: Pause on tab hidden, resume cleanly on visible

    **src/utils/coordinates.js** -- `toCanvasPixels(normalizedX, normalizedY, canvasWidth, canvasHeight)` and `toNormalized(pixelX, pixelY, canvasWidth, canvasHeight)`
    - `toCanvasPixels`: returns `{ x: Math.round(normalizedX * canvasWidth), y: Math.round(normalizedY * canvasHeight) }`
    - `toNormalized`: returns `{ x: pixelX / canvasWidth, y: pixelY / canvasHeight }`
    - Both functions handle edge cases: clamp normalized values to [0,1] range
    - CANV-04: Percentage-based coordinate system
    - CANV-08: Integer rounding in toCanvasPixels

    All files use ES module `export` syntax (no CommonJS). No default exports -- use named exports only.
