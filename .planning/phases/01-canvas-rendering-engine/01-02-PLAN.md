  <action>
    Build the custom hook and React component that wire all engine modules together.

    **src/hooks/useCanvasEngine.js** -- `useCanvasEngine(canvasRef)`
    - Accepts a React ref to the canvas element
    - On mount (useEffect with [] deps):
      1. Get canvas from ref, get context with `{ alpha: false }`
      2. Call `detectRefreshRate()` (async -- await it). During detection, set a `calibrating` state ref to true
      3. After detection completes, set calibrating to false, store results in refs
      4. Create frame loop via `createFrameLoop(onFrame, detectedFrameDurationMs)`
      5. The `onFrame` callback: clears canvas with #1a1a2e fillRect, then draws a subtle frame counter text in muted gray (for visual proof the loop runs). Store latest frame data in a ref for the debug panel.
      6. Set up ResizeObserver on canvas.parentElement:
         - On resize: call `calculateAspectRatio(containerWidth, containerHeight, 4/3, 600, 450)`
         - Apply computed width/height to canvas CSS (canvas.style.width/height)
         - Call `setupCanvasDPI(canvas, ctx)` to re-apply DPI scaling
         - Store current dimensions and isBelowMinimum in refs
      7. Create visibility manager via `createVisibilityManager(frameLoop)`
      8. Start the frame loop
    - On unmount (cleanup): stop frame loop, disconnect ResizeObserver, remove visibility listener
    - Return an object with refs/values the component needs: `{ calibrating, engineData, isBelowMinimum }`
      - `engineData` is a ref updated each frame: `{ hz, frameDurationMs, frameCount, droppedFrames, dpr, canvasWidth, canvasHeight, fps }`
      - FPS calculated as a rolling average (track last 60 deltas, compute 1000/avgDelta)
    - CRITICAL: No React useState for frame-loop data. All mutable engine state lives in useRef or plain variables. Only use useState for `calibrating` (triggers one re-render when calibration completes) and `isBelowMinimum`.
    - CANV-05: ResizeObserver recalculates and re-applies DPI on every resize without restarting the frame loop

    **src/components/CanvasRenderer.jsx** -- `CanvasRenderer`
    - Renders a container div (flex, centered, full-height) with the canvas element inside
    - Container div has `aspect-ratio: 4/3` CSS and fills available space via `max-w-full max-h-full`
    - Canvas element gets `ref={canvasRef}`, class with `touch-action: none`, `-webkit-touch-callout: none`, `user-select: none`
    - Canvas has no visible border -- blends with #1a1a2e background
    - Letterbox bars are the container's parent background (also #1a1a2e) -- seamless blend per user decision
    - Calls `useCanvasEngine(canvasRef)` and passes `engineData` and `calibrating` to child overlays
    - While calibrating: show centered "Calibrating display..." text in muted color over the canvas area (no spinner/animation per research recommendation)
    - CANV-03: 4:3 aspect ratio maintained via container CSS + JS resize calculation

    **src/App.jsx** -- Update from placeholder to full-bleed canvas layout
    - Root: `min-h-screen bg-[#1a1a2e] flex items-center justify-center`
    - Renders `<CanvasRenderer />` as the sole content (fully immersive -- no headers, nav, status bars per user decision)
    - Import and render all overlay components (DebugPanel, OrientationWarning, SmallScreenWarning -- built in Task 2)
    - For now, just render CanvasRenderer. Overlay imports will be added in Task 2.
